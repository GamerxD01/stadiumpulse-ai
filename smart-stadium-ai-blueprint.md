> **StadiumPulse AI** is a single GenAI "brain" that sits behind four connected experiences — a Fan Companion, a Staff Copilot, an Organizer Command Center, and a Volunteer Assistant — turning raw stadium data (crowd cameras, sensors, transit feeds, ticket data) into real-time, multilingual, natural-language guidance for everyone inside a World Cup venue.

## 4. The Four Experiences (What You Actually Demo)

### 4.1 Fan Companion (mobile chat/voice)
- "Where's the nearest accessible restroom?" → routes with step-free path
- "How do I get back to Gate C from here in Hindi?" → auto-detected language, turn-by-turn text
- "Is it going to be crowded near Gate 5 in 20 minutes?" → pulls live density prediction, answers in plain language
- "Best way to get to the airport after the match?" → Transit GPT, live congestion-aware

### 4.2 Staff & Volunteer Copilot (tablet/kiosk)
- Real-time alert: *"Density at Gate 3 is 92% of safe capacity, rising. Suggested action: open overflow Gate 3B, redirect fans via signage path 2."* — generated by the LLM reasoning over the density feed, with a **confidence score and short rationale** (real-time decision support).
- One-tap "explain this alert" for new volunteers — GenAI turns operational jargon into plain instructions.

### 4.3 Organizer Command Center (web dashboard)
- Auto-generated **shift-change briefing**: summarizes last 4 hours of incidents, crowd trends, and open issues in 3 bullet points.
- End-of-day **sustainability report**: waste generated, energy anomalies, water usage — GenAI drafts the narrative summary from raw numbers.

### 4.4 Accessibility Concierge (voice-first mode)
- Screen-reader-friendly, voice-in/voice-out
- Prioritizes step-free, elevator, and low-sensory-zone routing
- Can be demoed as a "mode" within the Fan Companion rather than a 5th separate app — keeps scope tight.

## 5. Tech Stack (works well inside Antigravity)

Antigravity is Google's agentic IDE, so lean into a stack it handles well:

| Layer | Suggested Tech |
|---|---|
| LLM | Gemini (via API) or Claude API — tool-calling / function-calling enabled |
| Orchestration | LangChain / LangGraph or a lightweight custom agent router |
| Frontend (Fan + Staff) | React + Tailwind (fast to scaffold, judge-friendly UI) |
| Dashboard (Organizer) | React + Recharts/Chart.js for live-looking visualizations |
| Backend | FastAPI or Node/Express |
| Data | Mocked JSON/CSV "sensor feeds" (crowd density, transit, weather) — cron/interval script to simulate live updates |
| Vector store | Chroma or simple in-memory embeddings for venue KB/FAQ RAG |
| Multilingual | LLM-native translation (no separate translation API needed) — keep architecture simple |
| Deployment | Vercel/Render for frontend+backend, or fully local demo if judged live |

**Tip:** Don't wire up real IoT hardware — judges know it's a hackathon. Simulate sensor data convincingly (randomized-but-realistic feeds with occasional "incident spikes") so the copilot has something dramatic to react to during the live demo.

## 6. Demo Script (3–5 minutes, this is what actually wins)

1. **Hook (15s):** "Imagine 80,000 fans from 100 countries in one stadium. One system, one AI brain, keeps everyone safe, moving, and informed — in their own language."
2. **Fan Companion demo (60s):** Ask a navigation question in English, then a transportation question in Spanish — show live translation.
3. **Trigger a live incident (60s):** Manually spike the mock crowd sensor feed → Staff Copilot fires an alert with a recommended action and reasoning → show operator acting on it.
4. **Organizer dashboard (45s):** Show the auto-generated shift briefing and sustainability report appearing in real time.
5. **Close (15s):** Return to the requirement-coverage matrix slide — "8 requirements, 1 architecture, deployable today."

Judges remember the **incident-trigger moment** most — build that first, polish it most.
